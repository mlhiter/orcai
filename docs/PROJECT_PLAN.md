# orcai — MVP 项目计划（通用知识点）

## 1. 目标（MVP）
构建一个“输入知识点 → 生成结构化教程”的平台型产品原型，支持：
- **通用知识点**（不限 K8s）
- **教程结构化输出**（而非一次性问答）
- **可复用资产沉淀**（同一知识点可复用/更新）

## 2. 价值主张（MVP 级）
- 覆盖性：大知识点也能产出完整、多章节教程
- 准确性：关键结论可追溯来源（至少核心章节）
- 复用性：相同知识点生成内容可复用（非每次重生成）

## 3. 用户范围（MVP）
- 面向“学习者”
- 首期不做企业内训/付费/协作

## 4. MVP 范围（功能清单）
### ✅ 必做
1) **知识点输入**（单点）
2) **教程生成**（结构化模板输出）
3) **教程展示**（网页页面）
4) **资产复用**（同知识点再次请求 → 复用已生成教程）
5) **引用展示**（核心章节显示参考来源）

### ❌ 暂不做
- 学习进度、测评
- 多语言
- 复杂权限/团队协作
- 搜索与推荐系统

## 5. 资产层级（最小模型）
- Concept（术语/概念）
- Topic（知识点）
- Module（章节）
- Course（教程）
- Path（学习路径）
> MVP 重点产出：**Topic + Course**，但结构上保留层级扩展接口

## 6. 教程模板（MVP 版）
- 概览：一句话概念、适用范围、学习目标
- 前置知识：必备概念清单
- 章节结构（多章节）
- 每章固定结构：概念 → 场景 → 原理 → 误区 → 小结
- 实践示例（最小可执行）
- 总结

**最小规范（先实践再迭代）：**
- 大概念默认 ≥ 5 章
- 必须包含：目标 / 前置 / 章节 / 总结 / 练习

## 7. 流程（高层）
1) 用户输入知识点
2) 系统判定：大概念/小概念
3) 生成教程（结构化）
4) 保存为教程资产
5) 二次请求 → 直接复用或轻量更新

## 8. 技术框架（仅方向，不落细节）
- 前端：教程展示页 + 输入框
- 后端：生成服务 + 资产存储
- 模型：LLM 生成 + 引用检索
- 存储：教程资产（可版本化）

## 9. 里程碑
- M1：输入 → 教程生成 → 展示页面
- M2：资产复用（相同知识点直接复用）
- M3：引用展示（核心章节来源）

### 当前实现状态（2026-03-25）
- ✅ M1 已完成
- ✅ M2 已完成（含刷新接口幂等：同知识点仅保留一个 active 任务）
- ✅ M3 已完成（章节展示引用来源，缺省为空数组）

## 10. 评估指标（MVP）
- 覆盖性：大知识点能产出完整多章节教程
- 复用率：同知识点二次请求直接复用 ≥ 70%
- 准确性：核心章节来源覆盖 ≥ 70%
